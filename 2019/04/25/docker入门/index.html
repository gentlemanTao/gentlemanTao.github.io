运行容器

docker使用run命令来进行容器的运行。run命令提供了容器的创建到启动的功能。

  首先，告诉docker执行run命令。并指定-i和-t两个命令行参数。-i标志保证容器的STDIN（标准输入）是开启的，可以进行持久输入以保证交互式操作正常进行。-t告诉docker要为创建的容器分配一个伪终端，这样才可以进行交互式shell。-i -t是在命令行下创建一个能够交互的容器最基本的参数。

  其次，我们告诉docker使用centos镜像来构建容器。此处是以centos（不加版本号默认为latest版本）镜像为基础来构建启动容器。构建步骤：

  • Docker会检查本地是否存在centos镜像，如果不存在就去DockerHub（公有仓库）下载并保存到本地。
  • Docker在文件系统内部使用这个镜像创建一个新容器，该容器拥有自己的网络，Ip地址，以及一个用来和宿主机进行通信的桥接网络接口。

  最后，我们告诉docker容器要运行/bin/bash命令。当容器创建完毕之后，Docker就会执行容器中的/bin/bash命令，这时就可以看到容器内的shell。

退出容器

使用exit命令退出容器，当退出之后，容器已经停止运行。只有在指定的/bin/bash命令处于运行状态时，容器才会运行。当退出容器之后，/bin/bash停止运行，容器也随之停止运行。

操作容器

查看容器列表

列出正在运行的容器

docker ps只会列出正在运行的容器列表。
列出全部容器

docker ps指定-a标志时，会列出所有容器，包括正在运行和已经停止的。
docker ps指定-l标志时，列出最后一个运行的容器，无论其正在运行还是已经停止。
从ps命令的输出结果可以看到容器的一些信息：id，容器的镜像，容器最后执行的命令，创建时间，以及容器的退出状态。
有3种方式可以唯一指定容器：短UUid(如5666f1fd1a0e）、长uuid、名称（festive_fermi）。

容器命名

给容器命名

docker每次创建时会为容器分配一个随机的名称，若想指定容器的名称，需要使用--name标志来实现容器的命名。一个合法的容器名称只能包含以下字符：小写字母a~z、大写字母A-Z、数字0～9、下划线、圆点、横线（正则表示：[a-zA-Z0-9.-]）。

重新启动已经停止的容器

启动已经停止的容器

通过id启动已经停止的容器

也可以使用docker restart命令来重新启动一个容器。
进入正在进行的容器

docker重新启动的时候，会使用docker run命令时指定的参数来运行，因此容器重新启动后会运行一个交互式shell。此时可以使用docker attach命令重新进入容器。可以使用容器的名称、长id，短id。如果退出容器的shell，容器会再次停止运行。

创建守护式容器

使用-d标志表示docker会将容器放在后台运行。使用docker ps命令可以看到容器daemon_dave正在运行。

查看容器内部

获取守护容器的日志

使用docker logs获取容器内部的日志。也可以在后面加上-f参数监控docker内部的日志，与tail -f很类似。
跟踪守护容器的日志

需要跟踪容器的某一片段时，可以加上tail命令。例如：可以使用docker logs --tail -10 daemon_dave获取日志的最后10行内容，使用docker logs --tail 0 f daemon_dave命令来跟踪某个容器的最新日志。
使用-t标志为每条日志加上时间戳

docker日志驱动

docker默认的日志驱动是json-file，其他可选项包括syslog，syslog选项将禁用docker logs命令,并且将所有容器的日志输出都重定向到Syslog。在docker run时可以指定该选项。
在容器级别启动syslog

Syslog默认配置下，日志会输出到/var/log/messages
也可以将日志驱动选项置为none，这个选项将会禁用所有容器的日志，docker logs命令也被禁用。

在容器内部运行进程

docker可以使用docker exec命令在容器内部启动额外的进程。可以在容器内运行的进程有两种：后台任务和交互任务。后台任务在容器内运行且没有交互需求，交互任务在保持在前台运行。
在容器运行后台任务

-d表示需要运行一个后台进程，-d之后指定需要执行的容器和在容器内需要执行的命令。

**在容器内运行交互命令

和run命令类似，-i和-t标志创建了TTY（伪终端）并捕捉STDIN（持续输入）。这个命令在容器内创建了一个新的bash会话。

自动重启容器

docker在run时可以使用--restart标志让docker自动重启容器，不能与--rm一起使用。
自动重启容器

docker的重启策略

  • no，默认策略，在容器退出时不重启容器
  • on-failure，在容器非正常退出时（退出状态非0），才会重启容器
  • on-failure:3，在容器非正常退出时重启容器，最多重启3次
  • always，在容器退出时总是重启容器
  • unless-stopped，在容器退出时总是重启容器，但是不考虑在Docker守护进程启动时就已经停止了的容器

为on-failure指定count参数

最多重启3次

自动删除容器

docker在run时可以使用--rm标志让docker在容器停止退出时自动删除。

-rm选项不能与-d同时使用,也不能与--restart一起使用。
